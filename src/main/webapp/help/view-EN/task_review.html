Flow chart for Task to Run/Review

(3) Create Task

Depending on projectâ€™s type (Android and IOS), the automated test provide different test. In Android projects, it provides compatibility test (Monkey, SmartMonkey), automated script test (Appium, Robotium and Uiautomator) and traversal test; IOS projects provide compatibility test (Monkey) and automated script test (Appium).

a) Project Manager (Project Member) logs in to the system and opens the project in "Project" page and click on the "Task" menu.

b) In Task page, click "Create Task". There are three steps: entry the basic information, select the mobiles and submit the task. Refer to Section "Tasks" for details.

(4) Review Report

When the task is completed, the user can view the task information and result in Task page. See Section "Tasks" for details.

For the presentation and analysis of test reports, see sections "Report" and "Analyze Report".

Users can perform automated tests according to business processes. They can perform compatibility, automated script and traversal test, quickly and efficiently get test reports, and reduce management costs for mobiles and builds. Automated test allows users to perform multiple tests, locates problems, and fix problems in a timely manner, so it shortens the product release cycle.

The report provides users a comprehensive evaluation for App. For automated testing, the system provides IOS and Android environment. The IOS operating system provides compatibility testing and script testing. The Android operating system provides compatibility testing, script testing, and traversal testing. See Section "Tasks" for details.

a) Result Review

It includes the basic information, the performance average and the test pass rate.

Result Review

i) The information is to show task name, creation time, start time and end time, and task status. Status has five: Created, Waiting, Executed, Completed, and Canceled.

Task Information

ii) The performance average is the analysis of the performance details of all the mobiles. It includes startup, CPU, memory, flow, and power. Startup is time that mobile install App; CPU refers to the test App occupies the CPU proportion of equipment; memory refers to the test App runtime occupied by the size of the mobile memory; Flow is app occupied of the upstream flow and downstream flow; power refers to loss of electricity value.

Average of Performance

iii) The test pass rate is a result of the compatibility test, automated script test and traversal test results in a pie chart, making it more clear and intuitive to view the pass rate. Success means that the install, launch and uninstall are successful; failure means that any stage of the install, launch, and uninstall is failure.

Test pass rate

b) Compatibility, Automated script and Traversal test

It performs detailed analysis of the compatibility, automated script and traversal test, and records its installation, startup, operation and uninstallation. When a problem or error occurs, the system automatically captures its wrong location and the cause of the error. Refer to Section "Report" for details.

c) Performance report

Performance reports are performance data recording start, CPU, memory, traffic, and power during the device execution test phase. Performance reports help users to analyze their App from the performance, making the test more comprehensive.

d) Mobiles List

The mobiles list is the devices which execute task.

e) Mobile Report

The mobile report are a detailed test record of each device, including the performance of the device's install, launch and uninstall. User can view the test process through screenshots, install logs, logcat logs and video.
