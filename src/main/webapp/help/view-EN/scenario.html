2.1 Scenario

The system provides one-stop completion of functional testing, compatibility testing, reliability testing and other services, the typical scenarios include: project creation, project maintenance, task execution and view, and remote test. Each scene consists of one or more business operations, which are implemented and executed by different user roles.

Typical Business Scenario

(1) User roles and Identities

The administrator has permission to operate operations A and B, and staffs do not have this permission. The administrator adds the system user through A, and then associates the project manager with the project when B adds the item. The project manager adds and edits the project manager and members through C. Refer to Section "Permissions" for details.
