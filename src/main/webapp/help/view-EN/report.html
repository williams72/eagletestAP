1.4.6 Report

The automated test provides three types (compatibility test, automated script test and traversal test). When the user submits the task, the system will perform it according to the task information. After the test is completed, the test result will be analyzed statistically and displayed on the test report page.

The report is divided into result review, compatible test, automated script test, traversal test, performance report, and mobile list.

(1) Result Review

This displays three types of information: task information, the pass rate and failure rate of each test type, and the overall performance.

Result Review

(2) Compatibility Test

Compatibility test is the result of the analysis task. It analyzes the details and the test phase (install, launch and uninstall). For the failure of the test, there are the wrong positioning and accounting statistics.

(3) Automated Script Test

Automated Script Test is the result of the analysis for automated scripting. It analyzes the test details of the task, the test phase (installs, runs, and unloads). For the failure, it provides the wrong location of the failed mobiles.

Traversal Test is the result of the analysis task. It analyzes the details and the test phase (install, launch and uninstall).

(5) Performance Report

Performance report shows the performance of each mobile performance, including start time, CPU, memory, flow consumption and power consumption, and shows the worst of the five models.

(6) Mobile List

The mobile list page lists the mobiles that perform this task, and the statistics for each device are tested. The system provides the download to export the test report, allowing the user to view the details of a device. As shown below:

Click the "View" button on the mobile list page, the user can view the detailed test report of a mobile. On the device details page, you can view the details of each test type through screenshots, installation logs, LogCat logs, compatibility logs, performance details, and screen recordings.
