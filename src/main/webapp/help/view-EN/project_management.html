1.3.2 Project Management

Project Management currently provides Create Project and Project List features. Create Project means that an administrator can add a new project. For details, see the "Create Project" section on this page. The Project List is list all the projects that are maintained in the system, as shown:

Project Management page

1.3.2.1 Create Projects

An administrator has the permission to add a new project. When adding a project, project information is needed, including project name, keyword, project type, project manager, and project description, as below:

Create Project page

The project type is divided into Android and IOS. In Android project, the user can use the mobiles whose OS is android and must upload the build whose file extension is .apk. And the android project provides compatibility testing (Monkey, SmartMonkey), automated script testing (Appium, Robotium and Uiautomator) and traversal tests; IOS project uses IOS mobile and upload the build whose file extension is .ipa. And there are two types for test, compatible Testing (Monkey) and automated script testing (Appium). Refer to Section “Create Task” for details. 1.4.3 User Management An administrator can add, disable or enable a Staff user’s account, and view their information. An administrator also can maintain and manage the users who use the system.
