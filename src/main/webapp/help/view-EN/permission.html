1.1.2 Permissions

The software provides two types of user permissions: one is User Role, the other is User Identity. User Role is used to specify permission at system level and it has two levels: Administrator and Staff, User Identity is used to specify permissions at project operation level and it has two levels: Project Manager and Project Member. Only after an User Role is assigned, you can be associated to one of User Identity.

User Role

User Identity

Administrator has the highest level of system access. Administrator can operate all system functions, including device management, user management, log, project management, and so on.

Staffs have less privilege than the Administrator. When the initial creation, Staff only can do remote test.

Whether it is an Administrator or a Staff, only to be associated User Identity with a project, they have authority to operate project related functions:

1) Project Manager can do all test functions for an assigned project (Dashboard, Version, Build, Member, Create Task, Member Management, and Review Report).

2) Project member can do part of the test functions for an assigned project (Dashboard, Version, Build, Member, Create Task, and View Report).
