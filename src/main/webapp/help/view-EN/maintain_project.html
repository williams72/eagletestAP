2.3 Maintain Project

The scenario includes project member management (C), version management (D), build management (E), as shown in the following figure:

Flow chart for Maintain Project

(1) Add Member

This allows project managers to add project members and managers to give users the identity of the user (project manager and member). The specific steps are as follows:

a) The project manager logs in and opens the project in the ”Project” page which manage and maintain. Click the Member menu.

b) Select the added user and identity, then click the Add button to add a new member to the project. See Section "Members".

(2) Maintain Member

The project manager edits an existing member. The operation flow is as follows:

a) The project manager logs in and opens the project in the "Project" page which manage and maintain. Click the Member menu.

b) Select a project member or manager to switch identity.

(3) Add Version

Add version allows the project managers and members to add the version, easily to manage the build. For details, refer to Section "Versions". The operation flow is as follows:

a) Project Manager (Project Member) logs in to the system and opens the project in the "Project" page and click on the "Version" menu.

b) In Version page, enter the version name and status, add a new version.

(4) Maintain Version

It means that the project managers and members change version status. Status includes development, testing, released. Refer to Section "Versions" for details.

(5) Add Build

It is the project manager and members to upload app. It is convenient to manage and maintain, also reduces the time to upload. Refer to section "Builds" for details.

(6) Maintain Build

It means that the project managers and members download and delete existing build. Refer to Section "Builds" for details.
