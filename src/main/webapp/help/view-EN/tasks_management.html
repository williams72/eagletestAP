1.4.2 Tasks Management

The task management provides the user to submit automated test tasks and display task information. In this function, the user can use the build to run mobile phone testing. When the test is completed, the user can view the task results and device information.

Tasks Management

1.4.2.1 Create Task

The user creates an automated test task that takes three steps, including completing the basic information, selecting the test model, and submitting the task.

(1) Basic Information

The user entry the basic information for the automated test task, including the task name, selecting the build, pre-job settings, selecting the test type, setting the test conditions. Due to the different types of projects, the types of tests provided are different.

Create tasks in Android Project

a) Pre-Job Setting: Allows user to do something before test task running, such as reboot or set the network simulation (2G, 3G, high packet loss rate and high latency) when create a task. b) There are three test types for an Android project, they are Compatibility test (Monkey, SmartMonkey), Automated script test (Appium, Robotium and Uiautomator), and Traversal test.

i)Compatibility Test

Compatibility test refers to the selected mobiles on the test App installed, start, monkey, uninstall and other testing process. It provides users with the test results, screenshots, installation logs, logcat log, video and other records. It also captures start time, CPU, memory, flow, power and other performance data to help users find the problem.

Compatibility test is divided into two sub-types of compatibility, one is General compatibility test, the other is Smart compatible test. General compatibility test is the implementation of Monkey test, all area (pages) will be clicked randomly, to ensure no server issue happened on all pages. Smart compatibility test is also the implementation of Monkey test with snapshot of each operation.

ii) Automated Script Test

Automated script tests provide users with customizable scripting files for App. Automated script test supports three test frameworks, including Appium, Robotium and Uiautomator. iii) Traversal Test

Traversal test is based on certain principles and rules of the pages and components to create a tree structure. In accordance with the tree structure, it traverses all the original controls and pages for App, which covers a variety of path branches, traverses all the functional modules in all detectable pages to increase the coverage of test cases.
